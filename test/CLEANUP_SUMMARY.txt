================================================================================
MOST COMMON COMMANDS
================================================================================

# Preview what will be deleted (SAFE - nothing deleted)
node test/servicenow-test-cleanup.js --dry-run

# Clean everything (asks for confirmation)
node test/servicenow-test-cleanup.js --all

# Clean everything without confirmation
node test/servicenow-test-cleanup.js --all --force

# Clean specific test
node test/servicenow-test-cleanup.js --test-id SN-SIMPLE-001

# Clean by difficulty level
node test/servicenow-test-cleanup.js --level simple
node test/servicenow-test-cleanup.js --level medium
node test/servicenow-test-cleanup.js --level complex

# Get help
node test/servicenow-test-cleanup.js --help

================================================================================
WHAT GETS CLEANED UP
================================================================================

1. ALL TEST ARTIFACTS
   - Business rules, client scripts, script includes
   - REST APIs, Service Portal widgets
   - Flow Designer flows, Import Sets, Transform Maps
   - Scheduled jobs, ACLs, test files
   - Documentation files
   
2. EMPTY DIRECTORIES
   - Removes any empty directories left after file deletion
   
3. GIT CHANGES
   - Reverts all modified files (git reset --hard HEAD)
   - Removes all untracked files (git clean -fd)
   
RESULT: sn-tools project back to clean state ✨

================================================================================
SAFETY FEATURES
================================================================================

✓ DRY RUN MODE: Always preview deletions first (--dry-run)
✓ CONFIRMATION: Asks "Continue? (yes/no)" before deleting
✓ FORCE MODE: Skip confirmation with --force flag
✓ GRACEFUL: Non-existent files don't cause errors
✓ DETAILED OUTPUT: Shows exactly what was deleted
